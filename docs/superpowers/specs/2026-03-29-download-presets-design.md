# Download Presets Feature Design

**Goal:** Allow users to save and reuse common download configurations (format, quality, output directory, embed options) with one-click application from the DownloadDialog.

**Architecture:** New `download_presets` SQLite table with dedicated Rust command layer and Pinia store. Presets are managed from the Settings screen and applied from DownloadDialog via a dropdown.

**Tech Stack:** Rust (rusqlite, Tauri commands), Vue 3 Composition API, TypeScript, Pinia

---

## Data Layer

### SQLite Table

```sql
CREATE TABLE IF NOT EXISTS download_presets (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL UNIQUE,
  format TEXT NOT NULL,
  quality TEXT NOT NULL,
  output_dir TEXT NOT NULL,
  embed_thumbnail INTEGER NOT NULL DEFAULT 0,
  embed_metadata INTEGER NOT NULL DEFAULT 0,
  write_subs INTEGER NOT NULL DEFAULT 0,
  embed_subs INTEGER NOT NULL DEFAULT 0,
  embed_chapters INTEGER NOT NULL DEFAULT 0,
  sponsorblock INTEGER NOT NULL DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

Individual columns (not options_json) because presets have a fixed, small set of fields — column-per-field makes editing and reading straightforward. Advanced options (rate_limit, proxy, extra_args) are intentionally excluded; they remain global settings.

### Rust Model (`src-tauri/src/db/models.rs`)

```rust
#[derive(Debug, serde::Serialize, serde::Deserialize)]
pub struct Preset {
    pub id: i64,
    pub name: String,
    pub format: String,
    pub quality: String,
    pub output_dir: String,
    pub embed_thumbnail: bool,
    pub embed_metadata: bool,
    pub write_subs: bool,
    pub embed_subs: bool,
    pub embed_chapters: bool,
    pub sponsorblock: bool,
    pub created_at: String,
}
```

## Backend

### New file: `src-tauri/src/commands/presets.rs`

Four Tauri commands:

| Command | Args | Returns |
|---------|------|---------|
| `list_presets` | — | `Vec<Preset>` |
| `create_preset` | name, format, quality, output_dir, embed_* | `i64` (new id) |
| `update_preset` | id, name, format, quality, output_dir, embed_* | `()` |
| `delete_preset` | id | `()` |

All commands follow the existing scoped-lock pattern:
```rust
let db = state.db.lock().await;
queries::list_presets(&db).map_err(|e| e.to_string())
```

### DB queries added to `src-tauri/src/db/queries.rs`

- `list_presets(conn)` — SELECT all, ORDER BY created_at DESC
- `insert_preset(conn, name, format, quality, output_dir, embed_*)` — INSERT, error on duplicate name
- `update_preset(conn, id, name, format, quality, output_dir, embed_*)` — UPDATE by id
- `delete_preset(conn, id)` — DELETE by id

### Changes to existing files

- `src-tauri/src/db/schema.sql` — add `download_presets` table definition
- `src-tauri/src/db/mod.rs` — ensure table is created on `init_db`
- `src-tauri/src/db/models.rs` — add `Preset` struct
- `src-tauri/src/lib.rs` — `mod presets;` + register 4 commands in `generate_handler!`
- `src-tauri/capabilities/default.json` — add 4 command permissions
- `src-tauri/permissions/autogenerated/` — 4 new permission toml files

## Frontend

### TypeScript type (`src/types/index.ts`)

```typescript
export interface Preset {
  id: number
  name: string
  format: string
  quality: string
  output_dir: string
  embed_thumbnail: boolean
  embed_metadata: boolean
  write_subs: boolean
  embed_subs: boolean
  embed_chapters: boolean
  sponsorblock: boolean
  created_at: string
}
```

### New file: `src/stores/presets.ts`

Pinia store with:
- `presets: ref<Preset[]>`
- `fetchPresets()` — invoke `list_presets`
- `createPreset(payload)` — invoke `create_preset`, then fetchPresets
- `updatePreset(payload)` — invoke `update_preset`, then fetchPresets
- `deletePreset(id)` — invoke `delete_preset`, then fetchPresets

Invoke parameter naming follows existing camelCase convention (e.g. `embedThumbnail`, `outputDir`).

### DownloadDialog changes (`src/components/download/DownloadDialog.vue`)

Two additions at the top of the form:

1. **Preset dropdown** — `<select>` populated from `presetsStore.presets`. Selecting a preset hydrates format, quality, output_dir, and all embed toggles. Default option: "プリセットを選択…" (no-op).

2. **「プリセットとして保存」button** — opens an inline name input. On confirm, calls `presetsStore.createPreset()` with current form values. Shows error if name is duplicate.

No other changes to DownloadDialog logic.

### New file: `src/components/settings/PresetSettings.vue`

Section component added to the settings scroll page (after RuleSettings, separated by `<hr>`).

**Layout:**
- Header: "ダウンロードプリセット" + 「新規作成」button
- Preset list: card per preset showing name, format/quality badge, output_dir
- Each card: 「編集」button + 「削除」button (with confirm dialog)
- Edit mode: inline form within the card (same fields as DownloadDialog — format select, quality select, output_dir input, embed toggles)
- Create mode: same inline form in a new blank card at the top of the list

### `src/App.vue` changes

- Import `PresetSettings` component
- Add `<PresetSettings />` with `<hr>` after `<RuleSettings />`
- Call `presetsStore.fetchPresets()` in `onMounted`

## Data Flow

```
DownloadDialog
  → select preset from dropdown
  → form fields hydrated from Preset object
  → user downloads (preset values used as DownloadOptions fields)

DownloadDialog
  → user configures options
  → clicks 「プリセットとして保存」
  → enters name → presetsStore.createPreset()
  → dropdown updated

PresetSettings (in Settings)
  → fetchPresets on mount
  → edit card → updatePreset()
  → delete card → confirm → deletePreset()
```

---

## Error Handling

- Duplicate preset name on create: Rust returns error string → displayed inline below name input
- Duplicate name on update: same pattern
- Delete confirmation: browser `confirm()` dialog before invoking delete (consistent with existing delete patterns in ScheduleView and library)
- Missing yt-dlp / download errors: unrelated to presets, handled by existing download layer

---

## Out of Scope

- Preset import/export
- Per-URL preset assignment
- Preset ordering / drag-and-drop
- Advanced options (rate_limit, proxy, extra_args) in presets — remain global settings only
