# Download Presets Feature Design

**Goal:** Allow users to save and reuse common download configurations (format, quality, output directory, embed options) with one-click application from the DownloadDialog.

**Architecture:** New `download_presets` SQLite table with dedicated Rust command layer and Pinia store. Presets are managed from the Settings screen and applied from DownloadDialog via a dropdown.

**Tech Stack:** Rust (rusqlite, Tauri commands), Vue 3 Composition API, TypeScript, Pinia

---

## Data Layer

### SQLite Table

```sql
CREATE TABLE IF NOT EXISTS download_presets (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL UNIQUE,
  format TEXT NOT NULL,
  quality TEXT NOT NULL,
  output_dir TEXT NOT NULL,
  embed_thumbnail INTEGER NOT NULL DEFAULT 0,
  embed_metadata INTEGER NOT NULL DEFAULT 0,
  write_subs INTEGER NOT NULL DEFAULT 0,
  embed_subs INTEGER NOT NULL DEFAULT 0,
  embed_chapters INTEGER NOT NULL DEFAULT 0,
  sponsorblock INTEGER NOT NULL DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

Individual columns (not options_json) because presets have a fixed, small set of fields вҖ” column-per-field makes editing and reading straightforward. Advanced options (rate_limit, proxy, extra_args) are intentionally excluded; they remain global settings.

### Rust Model (`src-tauri/src/db/models.rs`)

```rust
#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct Preset {
    pub id: i64,
    pub name: String,
    pub format: String,
    pub quality: String,
    pub output_dir: String,
    pub embed_thumbnail: bool,
    pub embed_metadata: bool,
    pub write_subs: bool,
    pub embed_subs: bool,
    pub embed_chapters: bool,
    pub sponsorblock: bool,
    pub created_at: String,
}
```

## Backend

### New file: `src-tauri/src/commands/presets.rs`

Four Tauri commands:

| Command | Args | Returns |
|---------|------|---------|
| `list_presets` | вҖ” | `Vec<Preset>` |
| `create_preset` | name, format, quality, output_dir, embed_* | `i64` (new id) |
| `update_preset` | id, name, format, quality, output_dir, embed_* | `()` |
| `delete_preset` | id | `()` |

All commands follow the project's scoped-lock pattern (lock dropped before any await point):
```rust
let result = {
    let db = state.db.lock().await;
    queries::list_presets(&db).map_err(|e| e.to_string())?
};
Ok(result)
```

### DB queries added to `src-tauri/src/db/queries.rs`

- `list_presets(conn)` вҖ” SELECT all, ORDER BY created_at DESC
- `insert_preset(conn, name, format, quality, output_dir, embed_*)` вҖ” INSERT, error on duplicate name
- `update_preset(conn, id, name, format, quality, output_dir, embed_*)` вҖ” UPDATE by id
- `delete_preset(conn, id)` вҖ” DELETE by id

### Changes to existing files

- `src-tauri/src/db/schema.sql` вҖ” add `download_presets` table definition
- `src-tauri/src/db/mod.rs` вҖ” ensure table is created on `init_db`
- `src-tauri/src/db/models.rs` вҖ” add `Preset` struct
- `src-tauri/src/commands/mod.rs` вҖ” add `pub mod presets;`
- `src-tauri/src/lib.rs` вҖ” register 4 commands in `generate_handler!`
- `src-tauri/capabilities/default.json` вҖ” add 4 command permissions
- `src-tauri/permissions/autogenerated/` вҖ” 4 TOML files are **auto-generated** by the Tauri build system on `cargo build` / `tauri dev`; do not create manually

## Frontend

### TypeScript type (`src/types/index.ts`)

Field names are snake_case, matching serde's default serialization of the Rust struct. Note: invoke() call parameters use camelCase (e.g. `outputDir`, `embedThumbnail`) вҖ” these are distinct from the interface field names.

```typescript
export interface Preset {
  id: number
  name: string
  format: string
  quality: string
  output_dir: string       // snake_case: matches Rust serde output
  embed_thumbnail: boolean
  embed_metadata: boolean
  write_subs: boolean
  embed_subs: boolean
  embed_chapters: boolean
  sponsorblock: boolean
  created_at: string
}
```

### New file: `src/stores/presets.ts`

Pinia store with:
- `presets: ref<Preset[]>`
- `fetchPresets()` вҖ” invoke `list_presets`
- `createPreset(payload)` вҖ” invoke `create_preset`, then fetchPresets
- `updatePreset(payload)` вҖ” invoke `update_preset`, then fetchPresets
- `deletePreset(id)` вҖ” invoke `delete_preset`, then fetchPresets

Invoke parameter naming follows existing camelCase convention (e.g. `embedThumbnail`, `outputDir`).

### DownloadDialog changes (`src/components/download/DownloadDialog.vue`)

Two additions at the top of the form:

1. **Preset dropdown** вҖ” `<select>` populated from `presetsStore.presets`. Selecting a preset hydrates format, quality, output_dir, and all embed toggles. Default option: "гғ—гғӘгӮ»гғғгғҲгӮ’йҒёжҠһвҖҰ" (no-op).

2. **гҖҢгғ—гғӘгӮ»гғғгғҲгҒЁгҒ—гҒҰдҝқеӯҳгҖҚbutton** вҖ” opens an inline name input. On confirm, calls `presetsStore.createPreset()` with current form values. Shows error if name is duplicate.

No other changes to DownloadDialog logic.

### New file: `src/components/settings/PresetSettings.vue`

Section component added to the settings scroll page (after RuleSettings, separated by `<hr>`).

**Layout:**
- Header: "гғҖгӮҰгғігғӯгғјгғүгғ—гғӘгӮ»гғғгғҲ" + гҖҢж–°иҰҸдҪңжҲҗгҖҚbutton
- Preset list: card per preset showing name, format/quality badge, output_dir
- Each card: гҖҢз·ЁйӣҶгҖҚbutton + гҖҢеүҠйҷӨгҖҚbutton (with confirm dialog)
- Edit mode: inline form within the card (same fields as DownloadDialog вҖ” format select, quality select, output_dir input, embed toggles)
- Create mode: same inline form in a new blank card at the top of the list

### `src/App.vue` changes

- Import `PresetSettings` component
- Add `<PresetSettings />` with `<hr>` after `<RuleSettings />`

`fetchPresets()` is called in each component's own `onMounted` (`PresetSettings.vue` and `DownloadDialog.vue`), keeping `App.vue` clean. This is consistent with how `ScheduleView` manages its own data load.

## Data Flow

```
DownloadDialog
  вҶ’ select preset from dropdown
  вҶ’ form fields hydrated from Preset object
  вҶ’ user downloads (preset values used as DownloadOptions fields)

DownloadDialog
  вҶ’ user configures options
  вҶ’ clicks гҖҢгғ—гғӘгӮ»гғғгғҲгҒЁгҒ—гҒҰдҝқеӯҳгҖҚ
  вҶ’ enters name вҶ’ presetsStore.createPreset()
  вҶ’ dropdown updated

PresetSettings (in Settings)
  вҶ’ fetchPresets on mount
  вҶ’ edit card вҶ’ updatePreset()
  вҶ’ delete card вҶ’ confirm вҶ’ deletePreset()
```

## Error Handling

- Duplicate preset name on create: Rust returns error string вҶ’ displayed inline below name input
- Duplicate name on update: same pattern
- Delete confirmation: browser `confirm()` dialog before invoking delete (consistent with existing delete patterns in ScheduleView and library)
- Missing yt-dlp / download errors: unrelated to presets, handled by existing download layer

---

## Out of Scope

- Preset import/export
- Per-URL preset assignment
- Preset ordering / drag-and-drop
- Advanced options (rate_limit, proxy, extra_args) in presets вҖ” remain global settings only
